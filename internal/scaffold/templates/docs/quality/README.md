# Quality

QS-xxx: quality scenarios — non-functional requirements written so they can be verified.

A quality scenario states a stimulus, the environment, and a measurable response ("given X, when Y, then Z within N"). Vague qualities ("fast", "usable") are not scenarios until they are measurable. Every Cliewen change is assessed against the active scenarios before its PR; a scenario a CI gate can hold is wired into CI.
