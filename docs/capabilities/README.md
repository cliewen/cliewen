# Capabilities

One folder per capability — the middle anchor of the red thread. Every
capability folder holds exactly three files, **one primary consumer
each**: `README.md` for whoever must *understand* (what + why, goal
reference), `criteria.md` for the linter/tester that must *verify*
(only Gherkin with AC-IDs), `design.md` for the implementer who must
*build* (how, diagrams inline as Mermaid). A fourth file that cannot
answer "who reads it?" does not get in.
